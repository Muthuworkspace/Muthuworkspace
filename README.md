<details>
<summary><b>🩺 Skin Cancer Classification — Deep Learning Research</b> &nbsp;|&nbsp; Published Research Project &nbsp;|&nbsp; 2024–2026 &nbsp;|&nbsp; Coimbatore</summary>

<br/>

> `TensorFlow` `Keras` `EfficientNetB0` `Transfer Learning` `Cost-Sensitive Learning` `OpenCV` `Docker` `FastAPI`

- Addressed severe class imbalance using **inverse-frequency class weights** (0.61 benign / 2.75 malignant) — model penalises missed cancer detections more heavily during training
- Fine-tuned EfficientNetB0 (ImageNet pre-trained) for binary classification — achieved **93% validation accuracy on 11,720 dermoscopic images**
- Applied rotation, flipping, zooming augmentation + ReduceLROnPlateau scheduler to prevent overfitting and improve convergence
- Evaluated with confusion matrix, ROC-AUC, F1, precision, and recall — not just accuracy, because **false negatives cost lives**
- 📄 **Research published** — IJIRT, Vol. 12, Issue 11, ISSN: 2349-6002, April 2026, Paper ID: 199064

</details>

<details>
<summary><b>🏦 Loan Eligibility Prediction — End-to-End ML Pipeline</b> &nbsp;|&nbsp; Personal Project &nbsp;|&nbsp; 2024 &nbsp;|&nbsp; Remote</summary>

<br/>

> `Python` `Scikit-learn` `XGBoost` `Random Forest` `SMOTE` `GridSearchCV` `Streamlit` `Render`

- Applied SMOTE on a 70/30 imbalanced dataset to synthetically balance the minority class — significantly reduced false negatives in loan rejection decisions
- Benchmarked Random Forest vs XGBoost head-to-head; selected Random Forest for superior generalisation on unseen records
- Tuned `n_estimators`, `max_depth`, `min_samples_split` via GridSearchCV — **pushed accuracy from 66% baseline → 90% on test data**
- Built full data pipeline: median imputation for missing values + RobustScaler for outlier handling before any model saw the data
- Deployed a **real-time credit risk web app** on Render using Streamlit — non-technical users get instant eligibility decisions

</details>
